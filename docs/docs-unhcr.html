UNHCR Refugee Statistics

CLI flag: --stats unhcr · ~450 datasets (6 endpoints x 75 years) · UNHCR Population Statistics API · No authentication required

API Endpoints

Base URL: https://api.unhcr.org/population/v1

EndpointPathData
Population/population/Stock figures: refugees, asylum-seekers, IDPs, stateless
Asylum Applications/asylum-applications/Individual asylum applications by country
Asylum Decisions/asylum-decisions/Decisions: recognized, rejected, closed
Solutions/solutions/Durable solutions: return, resettlement, naturalisation
Demographics/demographics/Age/gender breakdown of displaced populations
UNRWA/unrwa/UNRWA-registered Palestine refugees

Reference endpoints: /countries/ (249 countries), /regions/ (6 regions), /years/ (1951-present)

Commands & Usage

Flag Compatibility

FlagSupportedNotes
--scope✗Not applicable — UNHCR has a single fixed API
--formats✗Not applicable — always downloads JSON data
--year-from✓Start year filter (default: 1951)
--year-to✓End year filter (default: latest available)
--concurrency✓Auto-capped at 1 (UNHCR rate limits aggressively)
--verify✓Checks that _data.json files exist on disk
--dry-run✓Preview without downloading
--retry-failed✓Retry datasets that failed due to 403/429 errors
--filter-path✓Filter by endpoint path (e.g., unhcr/population)
--filter-codes✓Filter by dataset codes (e.g., population_2023,asylum-applications_2022)
--folder-style✓display (default) or code

Quick Reference (All Flags)

# collect — all applicable flags for UNHCR
inter-collect --stats unhcr \
  --year-from <START_YEAR:int> \
  --year-to <END_YEAR:int> \
  collect \
  -o <OUTPUT_DIR> \
  -c <CONCURRENCY:int> \
  -d <DELAY:seconds> \
  --skip-existing | --no-skip-existing \
  --retry-failed \
  --filter-path <ENDPOINT_PATH:str> \
  --filter-codes <ENDPOINT_YEAR1,ENDPOINT_YEAR2,...> \
  --dry-run \
  --verify \
  --folder-style <display|code> \
  -v

# status — show collection progress
inter-collect --stats unhcr status -o <OUTPUT_DIR>

# tree — display endpoint/year hierarchy
inter-collect --stats unhcr tree --depth <DEPTH:int> -v

# rename — switch folder naming style
inter-collect --stats unhcr rename <display|code> -o <OUTPUT_DIR> --dry-run -v

Not applicable for UNHCR: --scope, --formats (silently ignored)

Note: --concurrency is auto-capped at 1 (sequential) due to strict rate limiting.

Year Filtering (--year-from / --year-to)

# All years (default: 1951-present, ~450 datasets)
inter-collect --stats unhcr collect

# Recent data only
inter-collect --stats unhcr --year-from 2020 --year-to 2023 collect

# From a year onward
inter-collect --stats unhcr --year-from 2015 collect

# Single year
inter-collect --stats unhcr --year-from 2023 --year-to 2023 collect

Download Datasets

# Download all endpoints, all years
inter-collect --stats unhcr collect

# Preview what would be downloaded
inter-collect --stats unhcr collect --dry-run

# Download specific endpoint+year combinations
inter-collect --stats unhcr collect --filter-codes population_2023,demographics_2023

# Retry previously failed datasets
inter-collect --stats unhcr collect --retry-failed

# Verify completed datasets have data files
inter-collect --stats unhcr collect --verify

Important: UNHCR concurrency is automatically capped at 1 regardless of what you pass with --concurrency. This is because the UNHCR API has strict undocumented rate limits.

Check Collection Status

inter-collect --stats unhcr status

View Category Tree

inter-collect --stats unhcr tree
inter-collect --stats unhcr tree --depth 3

Rename Folders

inter-collect --stats unhcr rename code --dry-run
inter-collect --stats unhcr rename display

Files Per Dataset

For each endpoint+year combination (e.g., population_2023), 2 files are created:

FileFormatDescription
population_2023_info.jsonJSONCatalog snapshot (endpoint, year, page count)
population_2023_data.jsonJSONComplete paginated data with all rows

The _data.json file contains all rows across all API pages, merged into a single JSON array with aggregate totals.

Rate Limiting

The UNHCR API has undocumented rate limits. The collector uses:

  • Sequential requests (concurrency: 1)
  • 2-second delay between requests
  • Exponential backoff on 429/5xx: 15s, 30s, 60s, 120s
